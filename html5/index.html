Getting Started with HTML5

HTML5 Document Structure start here

Boilerplate & semantic layout

Every HTML5 page starts with a simple <!DOCTYPE html> declaration, followed by the <html>, <head>, and <body> elements. 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My HTML5 Page</title>
  </head>
  <body>
    <header>Site header</header>
    <main>Main content</main>
    <footer>Footer</footer>
  </body>
</html>

Use <header>, <nav>, <main>, <section>, and <article> to clearly describe the purpose of each part of the page.

Text & Links content

Headings, paragraphs, lists, anchors

Headings communicate the outline of your content. Use one <h1> per page, then <h2>, <h3>, etc. for subsections. 

<h1>Main heading</h1>
<h2>Subheading</h2>
<p>A paragraph of text with a
  <a href="https://example.com">link</a>.
</p>

<ul>
  <li>Unordered</li>
  <li>List</li>
</ul>

Always make link text descriptive, e.g. “View documentation” instead of “click here”, to help both users and screen readers.

Forms & HTML5 Inputs interaction

Form structure and built‑in validation

HTML5 adds many new input types that improve user experience and provide basic client‑side validation without JavaScript. 

<form>
  <label>
    Email:
    <input type="email" name="email" required />
  </label>
  <label>
    Age:
    <input type="number" name="age" min="0" max="120" />
  </label>
  <button type="submit">Submit</button>
</form>
  • type="email" checks for a valid email pattern.
  • type="number", type="date", and others give mobile‑friendly controls.
  • required, min, max, and pattern give simple validation rules.
