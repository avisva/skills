Getting Started with Java

Java Overview start here

Platform, JVM & JDK, first program

Java is a statically typed, object-oriented language that compiles to bytecode, which runs on the Java Virtual Machine (JVM). You write source code (.java), compile it with the JDK (Java Development Kit) into .class files, and the JVM executes them on any platform that has a compatible runtime.

  • JDK: Compiler (javac) + tools + JVM.
  • JRE: JVM + core libraries (runtime only).
  • WORA – “Write once, run anywhere” with bytecode on the JVM.
public class HelloWorld {
  public static void main(String[] args) {
    System.out.println("Hello, Java!");
  }
}

Syntax & Data Types basics

Variables, types, operators, control flow

Java has primitive types (e.g. int, double, boolean) and reference types (objects, arrays). Control structures (if, loops, switch) are similar to C/C++ but with strict compile‑time type checking. 

int age = 30;
double price = 19.99;
boolean active = true;
char grade = 'A';
String name = "Alice"; // reference type

if (age >= 18) {
  System.out.println("Adult");
} else {
  System.out.println("Minor");
}

for (int i = 0; i < 3; i++) {
  System.out.println("i = " + i);
}
  • Use final for constants.
  • Arrays: int[] nums = {1, 2, 3}; or new int[10];
  • Enhanced loop: for (int n : nums) { ... }

Object-Oriented Programming core

Classes, objects, inheritance, interfaces

Java is centered around classes and objects. You encapsulate state (fields) and behavior (methods). Inheritance and interfaces are used to create hierarchies and shared contracts. 

class Animal {
  protected String name;

  public Animal(String name) {
    this.name = name;
  }

  public void speak() {
    System.out.println("Animal sound");
  }
}

class Dog extends Animal {
  public Dog(String name) {
    super(name);
  }

  @Override
  public void speak() {
    System.out.println(name + " says: Woof!");
  }
}

interface RunnableTask {
  void run();
}
  • Use extends for single inheritance between classes.
  • implements lets a class satisfy one or more interfaces.
  • @Override ensures methods correctly override parent declarations.

Collections & Generics essential

Lists, Sets, Maps, generic types

The Collections Framework provides ready‑made data structures. Generics offer type safety at compile time, so you avoid many ClassCastExceptions at runtime. 

import java.util.*;

List<String> names = new ArrayList<>();
names.add("Alice");
names.add("Bob");

Set<Integer> uniqueIds = new HashSet<>();
uniqueIds.add(101);

Map<String, Integer> ages = new HashMap<>();
ages.put("Alice", 30);
ages.put("Bob", 25);

for (String n : names) {
  System.out.println(n);
}

Choose List when order matters, Set when uniqueness matters, and Map for key–value lookups.

Exceptions & Error Handling runtime

Checked vs unchecked, try / catch / finally

Java uses exceptions for error handling. Checked exceptions must be declared or caught; unchecked exceptions (subclasses of RuntimeException) do not. 

try {
  int result = 10 / 0; // ArithmeticException
} catch (ArithmeticException ex) {
  System.out.println("Cannot divide by zero");
} finally {
  System.out.println("Always executed");
}

For I/O, the try‑with‑resources syntax automatically closes resources:

try (BufferedReader reader =
       new BufferedReader(new FileReader("file.txt"))) {
  String line = reader.readLine();
} catch (IOException e) {
  e.printStackTrace();
}

J2EE / Jakarta EE Fundamentals enterprise

Servlets, JSP, JDBC, layers overview

J2EE / Jakarta EE defines a set of APIs and components for building large‑scale enterprise applications.

  • Servlets – Java classes that handle HTTP requests and responses.
  • JSP – JavaServer Pages: HTML templates with embedded Java (largely replaced by newer view technologies).
  • JDBC – Low‑level database access API.
  • JPA – ORM standard for mapping entities to database tables.
@WebServlet("/hello")
public class HelloServlet extends HttpServlet {
  @Override
  protected void doGet(HttpServletRequest req,
                       HttpServletResponse resp)
                       throws IOException {
    resp.getWriter().println("Hello from servlet");
  }
}

In modern projects you will very often see Spring Boot used with Java instead of a classic application server, but the core ideas above still apply.
